Sharing your Swiftlint config file among projects

2019-10-31
#swiftlint  #ios  #linter  #cocoapods 

At Teamwork iOS team we believe code has to be consistent throughout projects. So we have decided upon (and documented in a fairly long markdown doc) our code style, something we all agree and (sometimes) like. The good part is that now all code looks similar, which makes it easier for the eye/brain to spot parts of the code. Also, no more time wasted debating whether we’ll use tabs or spaces, vertical spacing or if opening curly braces should be placed in its own line or at the end.

It’s really nice to have everything documented, because you have a reference, but it’s even better if Xcode just point you errors in your code style. And that’s exactly what a linter does, in this case we use SwiftLint. So while we’re building errors and warnings pop-up and you can fix the style of the code. Our SwliftLint file has more than 200 rules, and we wanted to use it in all our projects. And now the fun times start.

The simplest way to do this is just copy over the .swiftlint.yml file to the new project. But you know what will happen next. You’ll update the Swiftlint config file in one project, forget in another and end up with several different Swiftlint config files. So much for “consistent”.

# needs a gem install cocoapods-keys  
plugin 'cocoapods-keys', {
  :project => "Launchpad-iOS",
  :keys => [
    "TeamworkGithubToken"
]}

# After installing all pods, we get the canonical SwiftLint config file from Projects
post_install do |installer|
  token = `pod keys get TeamworkGithubToken Launchpad-iOS` 
  download_file_from_github(
    "https://raw.githubusercontent.com", 
    "/Teamwork/projects-ios/develop/.swiftlint.yml", 
    token.strip, 
    ".swiftlint.yml"
  )
end

# downloads a file from a private GH repo
# base_url: should be something like https://raw.githubusercontent.com
# file_path: path to the file we need to download
# github_token: token to access private repos
# output_file_name: generated file name
def download_file_from_github(base_url, file_path, github_token, output_file_name)
  puts "Downloading file " + file_path + " from Github..."

  require 'net/http'
  require 'uri'

  uri = URI.parse( base_url + file_path )
  request = Net::HTTP::Get.new(uri)
  request["Authorization"] = "token " + github_token
  request["Accept"] = "application/vnd.github.v3.raw"

  req_options = {
    use_ssl: uri.scheme == "https",
  }

  response = Net::HTTP.start(uri.hostname, uri.port, req_options) do |http|
    http.request(request)
  end

  open(output_file_name, "wb") do |file|
    file.write(response.body)
  end

  puts "ðŸ’¾ Wrote file " + output_file_name
end
